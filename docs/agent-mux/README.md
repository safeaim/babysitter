# agent-mux docs

This section has three tracks:

- [Reference](reference/README.md) — the canonical API and behavior contract for `@a5c-ai/agent-mux`
- [Tutorials](tutorials/README.md) — task-oriented walkthroughs and usage guides
- [Archive](archive/README.md) — historical comparisons, design proposals, research, and superseded planning material

## Source of truth

`reference/` is the normative source of truth for current `@a5c-ai/agent-mux` behavior.

If a tutorial or archived document conflicts with a reference page, the reference page wins.

## Start here

- Package users: start with [Reference](reference/README.md)
- First-time users: start with [Getting Started](tutorials/getting-started.md), then confirm details in [Reference](reference/README.md)
- Historical context, parity notes, and exploratory design work: see [Archive](archive/README.md)

## Publishing contract

- `reference/` defines current behavior and API surface
- `tutorials/` is discoverable but non-normative
- `archive/` is preserved for context and is not source-of-truth

The main docs site consumes these markdown files directly. Keep the track boundaries explicit.
