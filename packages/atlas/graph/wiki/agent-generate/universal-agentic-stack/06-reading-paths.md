# Reading Paths

Use one of these short paths if you do not want to read the entire section linearly.

## I am evaluating a product claim

1. Read [`00-orientation.md`](./00-orientation.md)
2. Read [`04-worked-examples.md`](./04-worked-examples.md)
3. Scan [`07-comparison-matrix.md`](./07-comparison-matrix.md)
4. Use [`08-review-template.md`](./08-review-template.md)
5. Confirm details in [`layers.md`](./layers.md)

## I am designing an agent product

1. Read [`00-orientation.md`](./00-orientation.md)
2. Read [`01-builder-fit.md`](./01-builder-fit.md)
3. Read [`05-common-confusions.md`](./05-common-confusions.md)
4. Use [`03-placement-checklist.md`](./03-placement-checklist.md) against your own design

## I am debugging architecture confusion in a team

1. Start with [`05-common-confusions.md`](./05-common-confusions.md)
2. Use [`04-worked-examples.md`](./04-worked-examples.md) to show familiar shapes
3. Keep [`02-landscape-diagram.md`](./02-landscape-diagram.md) open as the shared reference

## I only want the shortest useful route

1. Read [`README.md`](./README.md)
2. Read [`00-orientation.md`](./00-orientation.md)
3. Read [`04-worked-examples.md`](./04-worked-examples.md)
4. Skim [`07-comparison-matrix.md`](./07-comparison-matrix.md)
5. Open [`09-faq.md`](./09-faq.md) if one question is still blocking you

## I need graph-grounded detail

1. Read [`layers.md`](./layers.md)
2. Open [`source-map.md`](./source-map.md)
3. Jump from there into the underlying graph records
