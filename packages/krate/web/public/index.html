a5c.ai Krate · Kubernetes-native forge

Krate MVP surface

Krate models repositories, pull requests, CI, policy, webhooks, and GitOps-transparent UI flows as Kubernetes-style resources.

Control Plane

CRDs stay in etcd; high-cardinality PR, issue, CI, review, and webhook records live behind the aggregated API/Postgres boundary.

Data Plane

Git smart-HTTP/SSH is backed by Gitea with branch/ref policy enforcement.

Identity & Policy

OIDC users map to Kubernetes users/groups; RBAC and admission own authorization and audit/enforce rollout.

Excellent Flows

  • Review PRs
  • Debug CI
  • Configure runners
  • Replay webhooks
  • Roll out policy
  • Share triage views
