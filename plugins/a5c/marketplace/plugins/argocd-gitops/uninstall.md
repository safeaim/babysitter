# argocd-gitops — Uninstall Instructions

## Step 1: Interview the User

Ask what to remove:

1. **CI only** — Remove the argocd workflow, keep Applications and manifests
2. **Manifests only** — Remove `gitops/` tree (Applications stay live until Argo CD notices them gone)
3. **Everything** — Delete Applications from cluster, remove manifests and workflow
4. **Selective** — Remove specific Applications or an entire environment

**Warning**: Deleting an Application with the `resources-finalizer.argocd.argoproj.io` finalizer cascades to deleting its managed resources in the cluster. Decide whether to preserve workloads before removing.

## Step 2: Decide on Cluster Cleanup

### Preserve workloads

Edit each Application and remove the finalizer before deletion:

```bash
kubectl patch application -n argocd frontend-prod \
  -p '{"metadata":{"finalizers":null}}' --type merge
```

Then delete (workloads remain; Argo CD just stops tracking them):

```bash
kubectl delete application -n argocd frontend-prod
```

### Cascade delete (remove managed resources)

```bash
kubectl delete application -n argocd frontend-prod
```

## Step 3: Remove Root Application

```bash
# Preserve children:
kubectl patch application -n argocd root -p '{"metadata":{"finalizers":null}}' --type merge
kubectl delete application -n argocd root

# Or cascade (deletes all children and their resources):
kubectl delete application -n argocd root
```

## Step 4: Remove ApplicationSets

```bash
kubectl delete applicationset -n argocd frontend-all-envs
```

Each Application generated by the set is cleaned up automatically.

## Step 5: Remove CI Workflow

```bash
rm -f .github/workflows/argocd.yml
```

## Step 6: Remove Manifests

```bash
rm -rf gitops/
```

## Step 7: Remove Argo CD CLI Token

1. Argo CD UI → Settings → Accounts → revoke the `ci` token
2. Repo → Settings → Secrets and variables → Actions → delete `ARGOCD_TOKEN` and `ARGOCD_SERVER` variable

## Step 8: Remove Install Scripts

Edit `package.json` and delete:

```json
{
  "scripts": {
    "argocd:sync:prod": "...",
    "argocd:diff:prod": "...",
    "argocd:bootstrap": "..."
  }
}
```

## Step 9: (Optional) Uninstall Argo CD Itself

Only if Argo CD was installed exclusively to support this plugin:

```bash
kubectl delete -n argocd -f https://raw.githubusercontent.com/argoproj/argo-cd/stable/manifests/install.yaml
kubectl delete namespace argocd
```

## Step 10: Remove Processes

```bash
rm -rf .a5c/processes/argocd-gitops
```

## Step 11: Remove from Registry

```bash
babysitter plugin:remove-from-registry --plugin-name argocd-gitops --project --json
```

## Notes

- The `AppProject` named `default` is usually shipped by Argo CD itself — do not delete unless you manage it here
- If other teams share the Argo CD instance, coordinate before deleting shared `AppProject` or RBAC
- ApplicationSet controller logs for deleted sets may keep events for a while; they self-prune
