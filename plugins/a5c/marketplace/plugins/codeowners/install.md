# codeowners — Install Instructions

Generate and maintain a `CODEOWNERS` file from git history, path patterns, and team mappings. Wires auto-reviewer-assignment via GitHub, validates the file on every PR, and keeps ownership up-to-date as code moves.

## Step 1: Interview the User

### Stage 1: Project Analysis

Before asking questions, analyze the project:
1. Check for existing `CODEOWNERS` at `.github/CODEOWNERS`, `docs/CODEOWNERS`, or root
2. Inspect git history: top authors per directory (`git log --format='%ae' --numstat`)
3. Identify team structure: infer from commit email domains, check for `TEAMS.md`, `OWNERS`
4. Detect monorepo structure (package boundaries are natural ownership units)
5. Check for existing PR templates and CI that reference reviewers
6. Summarize findings to the user

### Stage 2: Source of Truth

Ask:
- How should owners be inferred initially?
  1. **Git history** — Top committers per directory (default)
  2. **Manual mapping** — User provides team-to-path mapping
  3. **Hybrid** — Start from git, user reviews and edits before commit
- GitHub team handles (`@org/team-name`) or individual users (`@user`)? (default: teams)

### Stage 3: Granularity

Ask:
- Ownership granularity:
  - Package-level (default for monorepos)
  - Directory-level (default for flat repos)
  - File-pattern level (`*.sql`, `Dockerfile`)
- Minimum committers for a directory to get owners? (default: 2; below this falls back to a default owner)
- Fallback owner (catches orphan paths): `@org/maintainers` (default) or user-specified

### Stage 4: Policy

Ask:
- Require CODEOWNERS review for merge? (default: yes — configure branch protection)
- Auto-assign reviewers on PR open? (default: yes)
- Fail PR if `CODEOWNERS` is edited without a matching owner in the file? (default: yes)
- Schedule monthly ownership drift check? (default: yes)

## Step 2: Generate Initial CODEOWNERS

Create `scripts/gen-codeowners.sh`:

```bash
#!/usr/bin/env bash
set -euo pipefail

OUT=.github/CODEOWNERS
mkdir -p .github
{
  echo "# Auto-generated by codeowners plugin"
  echo "# Format: <path pattern>  <owners...>"
  echo
  echo "# Fallback"
  echo "*  @org/maintainers"
  echo
  echo "# Per-directory"
  for dir in $(git ls-tree -d --name-only HEAD); do
    top_authors=$(git log --format='%ae' -- "$dir" \
      | sort | uniq -c | sort -rn | head -3 | awk '{print $2}')
    if [ -n "$top_authors" ]; then
      handles=$(echo "$top_authors" | sed 's/@.*$//' | awk '{printf "@%s ", $1}')
      printf "/%s/  %s\n" "$dir" "$handles"
    fi
  done
} > "$OUT"

echo "Wrote $OUT"
```

```bash
chmod +x scripts/gen-codeowners.sh
./scripts/gen-codeowners.sh
```

Review and edit the output — **replace individual email-derived handles with GitHub team handles before committing**.

## Step 3: Structure the CODEOWNERS File

Final format at `.github/CODEOWNERS`:

```
# See https://docs.github.com/en/repositories/managing-your-repositories-settings-and-security/managing-repository-settings/about-code-owners

# Fallback
*                                @org/maintainers

# Infra
/.github/                        @org/platform
/infra/                          @org/platform @org/sre
/Dockerfile                      @org/platform

# Packages
/packages/sdk/                   @org/sdk-team
/packages/agent-mux/cli/                   @org/sdk-team
/packages/catalog/               @org/frontend

# Docs
/docs/                           @org/docs
*.md                             @org/docs

# Security-sensitive
/packages/sdk/src/auth/          @org/security @org/sdk-team
SECURITY.md                      @org/security
```

Rules (GitHub parser):
- Later patterns override earlier ones
- `/` prefix = repo root anchor
- Trailing `/` = directory
- Multiple owners = ANY-one can approve (unless `require_code_owner_reviews` + specific branch rule)

## Step 4: Install Validation

```bash
npm install -D codeowners-validator
```

Or via pre-built binary:

```bash
curl -L https://github.com/mszostok/codeowners-validator/releases/latest/download/codeowners-validator_linux_amd64.tar.gz | tar xz -C /usr/local/bin
```

Add to `package.json`:

```json
{
  "scripts": {
    "codeowners:check": "codeowners-validator --checks=files,owners,duppatterns,syntax"
  }
}
```

## Step 5: Create GitHub Actions Workflow

Create `.github/workflows/codeowners.yml`:

```yaml
name: CODEOWNERS
on:
  pull_request:
    paths: ['.github/CODEOWNERS']
  push:
    branches: [main]
  schedule:
    - cron: '0 6 1 * *'  # 1st of month, 06:00 UTC — drift check
jobs:
  validate:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
      - uses: mszostok/codeowners-validator@v0.7.4
        with:
          checks: 'files,owners,duppatterns,syntax'
          github_access_token: ${{ secrets.CODEOWNERS_TOKEN }}
          owner_checker_allow_unowned_patterns: 'false'

  drift-check:
    if: github.event_name == 'schedule'
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v6
        with: { fetch-depth: 0 }
      - name: Detect orphaned paths
        run: |
          ./scripts/gen-codeowners.sh
          git diff --exit-code .github/CODEOWNERS || \
            gh issue create \
              --title "CODEOWNERS drift detected" \
              --body "Auto-detected ownership drift. Please review." \
              --label "governance"
        env: { GH_TOKEN: '${{ secrets.GITHUB_TOKEN }}' }
```

**Note**: `CODEOWNERS_TOKEN` must be a PAT with `read:org` scope to validate team handles.

## Step 6: Configure Auto-Reviewer Assignment

GitHub automatically requests review from CODEOWNERS when a PR touches their files — provided:

1. The file is named `.github/CODEOWNERS` (or `docs/CODEOWNERS` / root)
2. Owners are valid GitHub users/teams with repo access
3. Repo setting: **Require review from Code Owners** is checked in branch protection

Enable via GitHub UI or the `branch-protection` plugin.

## Step 7: Add Pre-commit Hook

For Node:

```json
{
  "lint-staged": {
    ".github/CODEOWNERS": ["codeowners-validator --checks=syntax,duppatterns"]
  }
}
```

For Python:

```yaml
repos:
  - repo: local
    hooks:
      - id: codeowners-validator
        name: codeowners-validator
        entry: codeowners-validator --checks=syntax,duppatterns
        language: system
        files: ^\.github/CODEOWNERS$
        pass_filenames: false
```

## Step 8: Document Ownership in README

Add to `README.md`:

```markdown
## Code Ownership

See [`.github/CODEOWNERS`](.github/CODEOWNERS) for the full mapping.

To propose an ownership change, open a PR editing `.github/CODEOWNERS` and tag at least one current owner of the affected path.
```

## Step 9: Register Plugin

```bash
babysitter plugin:update-registry --plugin-name codeowners --plugin-version 1.0.0 --marketplace-name marketplace --project --json
```

## Step 10: Verify Setup

1. `.github/CODEOWNERS` exists and validates with no syntax errors
2. Opening a test PR auto-requests review from the owners of modified paths
3. Workflow runs and passes on a CODEOWNERS edit
4. All team handles resolve (no missing-team errors)
5. Fallback `*` rule covers any file not otherwise matched
6. Drift-check workflow is scheduled monthly
