# feature-flags — Configuration

## 1. Configuration Options Table

| Option | Values | Default | Where |
|--------|--------|---------|-------|
| `provider` | `openfeature`, `launchdarkly`, `unleash`, `flagsmith`, `configcat` | `openfeature` | `src/flags/client.*` |
| `namingPattern` | regex | `^[a-z0-9-]+$` | `flags/schema.json` |
| `flagTypes` | `release`, `experiment`, `ops`, `permission` | all four | `flags/schema.json` enum |
| `maxLifetime.release` | duration | `60d` | stale-flags script |
| `maxLifetime.experiment` | duration | `90d` | stale-flags script |
| `maxLifetime.ops` | duration | `never` | stale-flags script |
| `failOnUndeclared` | `on`, `off` | `on` | audit-flags script |
| `failOnStale` | `on`, `off` | `on` | workflow schedule job |
| `requireOwner` | `on`, `off` | `on` | schema `required` |
| `requireJiraTicket` | `on`, `off` | `off` | schema `required` |
| `auditSchedule` | cron | `0 9 * * 1` | workflow `schedule.cron` |
| `prodApprovalRequired` | `on`, `off` | `on` | provider-side policy |

## 2. Switch Provider

Edit `src/flags/client.ts`:

```typescript
import { OpenFeature } from '@openfeature/server-sdk';
import { UnleashProvider } from '@openfeature/unleash-provider';

await OpenFeature.setProviderAndWait(
  new UnleashProvider({
    url: process.env.UNLEASH_URL!,
    appName: 'my-app',
    clientKey: process.env.UNLEASH_API_TOKEN!,
  })
);
```

Update the corresponding secret in CI.

## 3. Adjust Naming Pattern

Edit `flags/schema.json`:

```json
"key": { "type": "string", "pattern": "^[a-z]+(\\.[a-z0-9-]+)+$" }
```

This enforces dotted-namespace keys like `checkout.new-flow`.

## 4. Change Expiration Policy

Edit `scripts/stale-flags.mjs` to add per-type windows:

```javascript
const maxAge = {
  release: 60,
  experiment: 90,
  ops: Infinity,
  permission: Infinity,
};
const ageDays = (Date.now() - new Date(f.createdAt)) / 86400000;
if (ageDays > maxAge[f.type]) { /* flag as stale */ }
```

## 5. Require a Jira Ticket per Flag

Edit `flags/schema.json`:

```json
"required": ["key", "type", "owner", "description", "createdAt", "jiraTicket"]
```

## 6. Soft-Launch Mode (warn-only)

Edit `.github/workflows/flags.yml`:

```yaml
jobs:
  audit:
    continue-on-error: true
```

Set a flip-to-error deadline in the workflow comment.

## 7. Ignore Specific Flags in Audit

Add to `scripts/audit-flags.mjs`:

```javascript
const IGNORE = new Set(['legacy-admin-flag', 'vendor-internal']);
const undeclared = [...used].filter((k) => !declared.has(k) && !IGNORE.has(k));
```

## 8. Integrate with Release-please

Auto-cleanup PRs for stale flags:

```yaml
- name: Open stale-flag PR
  if: failure()
  run: |
    gh pr create --title "chore(flags): cleanup stale flags" \
      --body "Auto-generated by stale-flags job" \
      --label "tech-debt"
```

## 9. Per-Environment Defaults

Edit `flags/flags.yaml`:

```yaml
flags:
  - key: new-dashboard
    defaults:
      dev: true
      staging: true
      prod: false
      canary: 0.10   # 10% rollout
```

Wire the SDK to read `NODE_ENV` and select the right default during bootstrap.

## 10. Cleanup Baseline Flags

Run the babysitter cleanup process:

```bash
babysitter run:create \
  --process-id feature-flags-cleanup \
  --entry .a5c/processes/feature-flags/cleanup.js#process \
  --prompt "Remove all flags past expiresAt, replace usages with the rolled-out branch, and open a PR" \
  --json
```

## 11. Change Audit Schedule

Edit `.github/workflows/flags.yml`:

```yaml
schedule:
  - cron: '0 3 * * *'   # daily 03:00 UTC
```
