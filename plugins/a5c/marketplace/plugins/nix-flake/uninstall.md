# nix-flake — Uninstall Instructions

## Step 1: Interview the User

Ask what to remove:

1. **Flake only** — Remove `flake.nix` and `flake.lock`, keep direnv and CI
2. **direnv only** — Remove `.envrc`, keep flake
3. **Everything** — Remove flake, direnv config, CI workflow, and pre-commit integration
4. **Selective** — Let the user pick

**Warning**: Removing the flake drops the project's reproducibility guarantee. Every contributor will fall back to their host toolchain. Confirm with the user.

## Step 2: Remove Flake Files

```bash
rm -f flake.nix flake.lock
rm -f shell.nix default.nix   # if present as legacy fallbacks
```

If flake inputs or outputs were referenced from other Nix files in the repo (e.g. `nixos-config/`), check and clean those first.

## Step 3: Remove direnv Integration

```bash
rm -f .envrc .envrc.local
rm -rf .direnv/
```

Remove the direnv-related `.gitignore` lines if desired.

If direnv and nix-direnv were installed system-wide purely for this project, optionally uninstall:

```bash
brew uninstall direnv nix-direnv
# or
nix profile remove direnv nix-direnv
```

Do not uninstall if other projects on the machine rely on direnv.

## Step 4: Remove CI Workflow

```bash
rm -f .github/workflows/nix.yml
```

If Nix steps were merged into an existing workflow, remove only those steps.

## Step 5: Remove Cachix Configuration (Optional)

If a Cachix token was set as a repo secret (`CACHIX_AUTH_TOKEN`), remove it via `Settings → Secrets and variables → Actions`.

If a Cachix cache was created specifically for this project, deprovision it at https://app.cachix.org.

## Step 6: Remove Nix Pre-commit Integration

If `git-hooks.nix` was used, nothing to uninstall — removing `flake.nix` already removed the hook definitions. Run:

```bash
pre-commit uninstall 2>/dev/null || true
rm -rf .pre-commit-config.yaml
```

Only remove `.pre-commit-config.yaml` if it was generated by `git-hooks.nix` (not hand-maintained).

## Step 7: Clean Local Nix Artifacts

```bash
rm -f result result-*
nix-collect-garbage -d    # optional: reclaim disk
```

`nix-collect-garbage -d` also removes old generations — only run if safe on the developer machine.

## Step 8: Uninstall Nix Itself (Optional)

Only if no other projects need it:

```bash
# Determinate installer
/nix/nix-installer uninstall

# Official installer — manual steps per
# https://nixos.org/manual/nix/stable/installation/uninstall.html
```

## Step 9: Restore Host Toolchain Documentation

Edit `CONTRIBUTING.md` / `README.md` and replace Nix bootstrap instructions with host-install instructions (nvm, pyenv, rustup, etc.) or re-install an alternative such as the `mise-asdf` plugin.

## Step 10: Remove from Registry

```bash
babysitter plugin:remove-from-registry --plugin-name nix-flake --project --json
```

## Notes

- `flake.lock` history remains in git history; to scrub it from all commits, use `git filter-repo` (beyond the scope of this uninstall)
- Nix store paths on developer machines are not removed automatically — `nix-collect-garbage -d` reclaims them
- CI history from the removed workflow remains in GitHub Actions run history
