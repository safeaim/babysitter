# release-please — Configuration

## 1. Configuration Options Table

| Option | Values | Default | Where |
|--------|--------|---------|-------|
| `releaseType` | `node`, `python`, `go`, `rust`, `java`, `php`, `ruby`, `terraform-module`, `simple` | `node` | `release-please-config.json` |
| `releaseBranch` | branch name | `main` | workflow `on.push.branches` |
| `includeComponentInTag` | `true`, `false` | `true` (manifest) / `false` (simple) | per-package config |
| `separatePullRequests` | `true`, `false` | `false` | root config |
| `versioningStrategy` | `default`, `always-bump-patch`, `always-bump-minor`, `service-pack` | `default` | per-package |
| `bumpMinorPreMajor` | `true`, `false` | `false` | per-package (0.x projects) |
| `bumpPatchForMinorPreMajor` | `true`, `false` | `false` | per-package |
| `plugins` | `node-workspace`, `cargo-workspace`, `linked-versions` | `[]` | root config |
| `draft` | `true`, `false` | `false` | per-package |
| `prerelease` | `true`, `false` | `false` | per-package |
| `labels` | array of labels | `['autorelease: pending']` | root config |
| `releaseLabels` | array | `['autorelease: tagged']` | root config |

## 2. Change Release Branch

Edit workflow:

```yaml
on:
  push:
    branches: [main, next]
```

And config:

```json
{
  "packages": {
    ".": { "target-branch": "next", "prerelease": true }
  }
}
```

## 3. Force a Specific Version

Add a commit with a `Release-As:` footer:

```bash
git commit --allow-empty -m "chore: release 2.0.0" -m "Release-As: 2.0.0"
```

## 4. Customize Changelog Sections

```json
{
  "changelog-sections": [
    { "type": "feat", "section": "🚀 Features" },
    { "type": "fix", "section": "🐛 Bug Fixes" },
    { "type": "perf", "section": "⚡ Performance" },
    { "type": "deps", "section": "📦 Dependencies", "hidden": false },
    { "type": "docs", "section": "📚 Documentation", "hidden": true }
  ]
}
```

## 5. Pre-release Channels

```json
{
  "packages": {
    ".": {
      "prerelease": true,
      "prerelease-type": "beta",
      "target-branch": "next"
    }
  }
}
```

Produces `v2.0.0-beta.1`, `v2.0.0-beta.2`, etc.

## 6. Linked Versions (Monorepo)

Force several packages to share a version number:

```json
{
  "plugins": [
    {
      "type": "linked-versions",
      "groupName": "core",
      "components": ["@a5c-ai/sdk", "@a5c-ai/cli"]
    }
  ]
}
```

## 7. Skip Labels

Skip a release PR by adding a label:

```bash
gh pr edit <number> --add-label "autorelease: snapshot"
```

## 8. Manifest Drift Repair

If `.release-please-manifest.json` gets out of sync with published tags:

```bash
# Set each package's current version manually
jq '.["packages/sdk"] = "1.4.2"' .release-please-manifest.json > .tmp && mv .tmp .release-please-manifest.json
git commit -am "chore: repair release-please manifest"
```

## 9. Customize PR Title/Body

```json
{
  "pull-request-title-pattern": "chore: release ${component} ${version}",
  "pull-request-header": "Release PR generated by release-please."
}
```

## 10. Run Locally (Dry Run)

```bash
npx release-please release-pr \
  --repo-url=owner/repo \
  --token="$GITHUB_TOKEN" \
  --config-file=release-please-config.json \
  --manifest-file=.release-please-manifest.json \
  --dry-run
```

## 11. Extra File Updaters

Update version references in non-standard files:

```json
{
  "extra-files": [
    "src/version.ts",
    { "type": "generic", "path": "README.md" },
    { "type": "json", "path": "custom.json", "jsonpath": "$.version" }
  ]
}
```

Each file receives `x-release-please-version` / `x-release-please-major` comments where updates are injected.
